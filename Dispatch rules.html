Dispatch

Dispatch is Resondible for:

The Dispatch board and the Traige board.
As well as answering phones
Assigning tickets out and Checking all IT staff calendars

How this is done:
  • New tickets land in Triage board and need to be processed ASAP
  • Triage processing sorts out spam and separates admin, sales, voice and support(IT) board tickets
  • Tickets to Voice board are set as Status: NEW but Tickets to IT board are set as Status: Unassigned (otherwise the user gets a second emailed response)
  • Some customers are co-managed so their tickets go to their customer board
  • Phone calls to office require a new ticket or existing ticket number before transferring call to support
  • Dispatcher must follow up on all red or orange tickets before end-of-day
  • Processing all tickets before 5pm is required.
Using Manage to Dispatch I.T. tickets
  1. Review description and set appropriate priority.
  2. Attach or Connect their related I.T equipment(desktop/laptopk/server) under configurations
  3. Set Type/Subtype and Item as appropriate to description of problem
  4. Assign available technician and see section below for special(red and orange) priority ticket processing
  5. If appropriate contact customer yourself
  6. Update summary of ticket to more closely align with customer Request
Bonus material and add on things
Follow up with tech assigned red tickets Page them(teams, cell phone call, etc.)to alert them and perform a clean "hand off" Coordinate between a tech travelling and customer as to when tech can go onsite. Add this scheduled entry to the ticket calendar
Any Red tickets for servers being down that are not automatically closed, investigate and report the current status. Follow up Notification by email(suggested wording)
